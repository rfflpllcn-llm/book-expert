## Goal

Extend the preprocessing pipeline and runtime cite tool to support bilingual books. When a book has an original + translated version, the agent can cite both side-by-side on demand.

## Key Design Decisions

- **Alignment is orthogonal** â€” the knowledge base, chunks, arcs are built from the original only. Alignment is a separate artifact consumed at runtime.
- **Translation on demand** â€” cite shows the original by default; `--lang it` adds the translation alongside.
- **Line-level alignment file** â€” bertalign produces sentence-level beads, but with `is_split=true` each JSONL line is one "sentence", so bead indices map directly to line indices. The stored artifact maps line IDs to line IDs, matching what the cite tool already works with.
- **Pre-split input** â€” each JSONL line is fed as-is to bertalign with `is_split=true` (matching pdfalign-aligner's approach). This avoids the lossy `clean_text()` â†’ `split_sents()` path where whitespace normalization and sentence splitting destroy the mapping back to source line IDs.
- **bertalign copied into project** â€” the bertalign package is copied from pdfalign-aligner into book-expert as a self-contained local package.

### 2. New preprocessing stage: `preprocessing/02a__alignment/align.py`

**CLI interface:**
```bash
python -m preprocessing.02a__alignment.align \
  --src data/voyage-fr.jsonl \
  --tgt data/voyage-it.jsonl \
  --out data/alignment-fr-it.jsonl \
  [--max-align 3] [--top-k 10] [--win 10] [--min-win-size 1] [--percent 0.15]
```

**Process:**
1. Load both JSONL files (`{"id": "...", "t": "..."}` format)
2. Join all `t` fields with `\n` â€” one line per JSONL entry
3. Feed to `Bertalign(src_text, tgt_text, is_split=True, ...)` â€” each JSONL line becomes one bertalign "sentence"
4. Call `align_sents()` to get beads: `([src_sent_indices], [tgt_sent_indices])`
5. Map bead indices directly to JSONL line IDs (index 0 â†’ first JSONL entry's `id`, etc.)
6. Write output JSONL

**Why `is_split=True`:** bertalign's `clean_text()` normalizes whitespace and drops blank lines, and `split_sents()` returns only sentence strings with no offsets. Using `is_split=False` would destroy the mapping back to source line IDs. With `is_split=True`, bead indices correspond 1:1 to input lines â€” the same approach pdfalign-aligner uses in production.

**Default bertalign parameters** (from pdfalign-aligner config.yaml):
- `max_align`: 3
- `min_win_size`: 1
- `percent`: 0.15
- `win`: 10
- `top_k`: 10
- `is_split`: true (preserves JSONL line boundaries)

**Output format** (`alignment-fr-it.jsonl`):

Uses generic keys `src_lines` / `tgt_lines` so the same format works for any source/target language pair:

```json
{"src_lines": [77, 78], "tgt_lines": [82, 83, 84], "type": "2-3"}
{"src_lines": [79], "tgt_lines": [85], "type": "1-1"}
{"src_lines": [80, 81], "tgt_lines": [86], "type": "2-1"}
```

### 3. book.yaml schema extension

Add optional `translations` section:

```yaml
source_text:
  file: data/voyage-fr.jsonl
  format: jsonl
  line_prefix: FR
  line_column: id
  text_column: t

translations:
  it:
    file: data/voyage-it.jsonl
    format: jsonl
    line_prefix: IT
    line_column: id
    text_column: t
    alignment: data/alignment-fr-it.jsonl
```

The translations schema mirrors `source_text` fields (`format`, `line_column`, `text_column`) so `lib/cite.py` can load translation lines through the same `_load_lines_jsonl` path.

Extensible to multiple translations. No changes to `arcs`, `characters`, `themes`, `chunks`, or any other existing section.

**Ownership:** the `translations` section is NOT generated by stage 04 â€” it is added by the alignment workflow (stage 02a) or manually. Stage 04 only generates the sections it already knows about (`source_text`, `chunks`, `arcs`, `characters`, `themes`, `knowledge_base`).

### 4. `lib/cite.py` â€” add `--lang` flag

**Current CLI:** `python -m lib.cite . <start> <end>`
**New CLI:** `python -m lib.cite . <start> <end> [--lang it]`

**Behavior with `--lang it`:**
1. Load FR lines as usual
2. Load `alignment-fr-it.jsonl` (path from book.yaml `translations.it.alignment`)
3. Find all alignment beads where `src_lines` overlaps with the requested range
4. Collect the union of `tgt_lines` from matching beads
5. Load those IT lines from `voyage-it.jsonl` (path from book.yaml `translations.it.file`)
6. Return both, with gap-aware formatting for translation lines:

```
> Â«Notre vie est un voyage / Dans l'hiver et dans la NuitÂ» (FR2â€“FR3)
> Â«La nostra vita Ă¨ un viaggio / Nell'inverno e nella NotteÂ» (IT5â€“IT7)
```

When many-to-many beads produce non-contiguous target lines (e.g. IT5, IT7 with no IT6), format as separate contiguous ranges joined by ` + `:

```
> Â«...Â» (IT5 + IT7)
```

This requires `format_citation()` to detect gaps in the line ID sequence and split into contiguous groups, rather than blindly labeling `firstâ€“last`.

**Without `--lang`:** identical to current behavior. Books without a `translations` section are unaffected.
